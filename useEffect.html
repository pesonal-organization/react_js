Question 3❓

Q: what is side effects in reactJs?

✅ Answer

If myFunction() does something outside rendering (like console.log, API call, setting timer, changing DOM, etc.) → ✅ that IS a side effect.

💡 Example


       
        function App() {
  useEffect(() => {
    console.log("I ran a function!"); // side effect
  }, []);

  return <h1>Hello World</h1>;
}

Question 6❓

Q: what is useEffect( )?

✅ Answer

👉 useEffect is a React Hook that lets you run code when your component renders or when something changes.
useEffect is used for side effects like fetching data, updating the DOM, running code when something changes, or cleaning up resources.

💡 Example


       
        🔹 Uses of useEffect

Run code when the component first shows up (like page load).

useEffect(() => {
  console.log("Component mounted");
}, []);


Fetch data from an API

useEffect(() => {
  fetch("https://api.example.com/data")
    .then(res => res.json())
    .then(data => console.log(data));
}, []);


Update something outside React (like page title or console)

useEffect(() => {
  document.title = "New Title";
}, []);


Run code when a value changes

useEffect(() => {
  console.log("Count changed");
}, [count]); // runs when count changes


Set up and clean up (timers, event listeners, etc.)

useEffect(() => {
  const timer = setInterval(() => console.log("Tick"), 1000);

  return () => clearInterval(timer); // cleanup
}, []);

Question 7❓

Q: what is cleanUp in useEffect?

✅ Answer

i make a useeffect for watch than i create a variable and i keep the setinterval in my variable that i have been created in useeffect, now after setinterval inside use effect i create a return call back function and in blocks of function i write clearinterval and inside its parameter i pass my variable name, it is cancellation process

💡 Example


      
         import { useState, useEffect } from "react";

function Watch() {
  const [time, setTime] = useState(0);

  useEffect(() => {
    // start interval
    const timer = setInterval(() => {
      setTime((prev) => prev + 1);
    }, 1000);

    // cleanup (cancellation process)
    return () => {
      clearInterval(timer);
    };
  }, []); // empty array → runs only once

  return <h1>{time} seconds</h1>;
}

Question 8❓

Q: order of useState, functions,useEffect and return?

✅ Answer

first i create useState,than function that deals to useState ect , than useEffect and than return and inside return i show data on ui .
👉 useState → function → useEffect → return (UI).

💡 Example


       
        import { useState, useEffect } from "react";

function MyComponent() {
  // 1️⃣ useState → to store data
  const [count, setCount] = useState(0);

  // 2️⃣ function → to update state or do something
  function increase() {
    setCount(count + 1);
  }

  // 3️⃣ useEffect → to handle side effects
  useEffect(() => {
    console.log("Count is now:", count);

    // cleanup (optional)
    return () => {
      console.log("Cleanup before next effect or unmount");
    };
  }, [count]); // runs when 'count' changes

  // 4️⃣ return → to show UI
  return (
    <div>
      <h1>{count}</h1>
      <button onClick={increase}>Increase</button>
    </div>
  );
}

export default MyComponent;

Question 12❓

Q: if i dont pass dependencies[ ] in useEffect what will be happen?

✅ Answer

the code or console that we have declared inside { } run again and again when page or state changes.
useEffect(()=>{}) → runs after every render.
useEffect(()=>{}, []) → runs only once.
useEffect(()=>{}, [count]) → runs when count changes.

💡 Example


       
        import { useState, useEffect } from "react";

function App() {
  const [count, setCount] = useState(0);

  useEffect(() => {
    console.log("hello");
  });

  return (
    <div>
      <h1>{count}</h1>
      <button onClick={() => setCount(count + 1)}>Increase</button>
    </div>
  );
}


What happens here?

Page load → "hello" logs.

Click button → count changes → component re-renders → "hello" logs again.

Every click = "hello" logged again.
