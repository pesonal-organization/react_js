Question 5❓

Q: what is App.jsx?

✅ Answer

app.jsx is the main components of our application.
think of it is the main room of house where we can keep everty thing
Inside app.jsx we keep

  • navbar
  • hero section
  • cards
  • footer
  • others required components

    • main.jsx runs first

    main.jsx calls app.jsx

    app.jsx shows every thing on the screen

    app.jsx is called the root component of application
    usually others all components are added here

    💡 Example

    function App() { return ( <div> <Navbar /> <Hero /> <Footer /> </div> ) }

    Question 9❓

    Q: what is ESLint?

    ✅ Answer

    ESLint is a tool that checks our JavaScript/React code.
    It helps us find and fix mistakes(like missing semicolons,unused variables or wrong code style).
    It makes our code clean, consitent and error-free.

    Question 10❓

    Q: what is eslint.config.js?

    ✅ Answer

    eslint.config.js is the rulebook that tells Eslint how to check our code.
    eslint.config.js is the file where we set the rules for checking.
    Example rules
    should we use semicolons or not
    should we use double quotes or single quotes
    should it warn if we don't use a variable.

    Question 15❓

    Q: Difference among node_modules, package.json and package-lock.json?

    ✅ Answer

    1.node_modules
    A big storage box
    contains all the actual libraries(React, vite etc) that projects uses.
    2.package.json
    A shopping list
    says which libraries project needs(react,react-dom etc)
    3.pakage-lock.json
    A receipt
    Records the exact versions of installed libraries.
