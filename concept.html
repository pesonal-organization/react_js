Question 14‚ùì

Q: show dynamic id of an object on another page:

‚úÖ Answer

In the first component, we call an API and store the response data in state using useState.
After getting the data, we display it on the screen using the map() method.
Inside the mapped data, we create a View User button for each list item.
On the button, we use the onClick event and pass a callback function.
Inside this callback function, we call the viewUser function and pass the current user data as a parameter.
Now we create the viewUser function and receive the data in its parameter.
Inside the viewUser function, we use the navigate() function from react-router-dom.
In the navigate() function, we use backticks and pass the dynamic path along with the user id like /newpage/${data.id}.
When the button is clicked, React Router navigates to a new page with a dynamic URL.
Now we go to the page where we want to show the dynamic id.
On this page, we import the useParams() hook from react-router-dom.
We destructure the id directly from useParams().
The useParams() hook reads the dynamic value from the URL.
we go in routes and the route the page where we have to show dynamic id write like path='/page:id'
Finally, we display the dynamic id in JSX using {id}.
In this way, dynamic routing works in React and the dynamic id is successfully shown on the page.

Question 20‚ùì

Q: useMemo hook and example:

‚úÖ Answer

If we have a function and we have two states, normally when any state changes the function runs again. But we don‚Äôt want the function to run on every state change because this affects performance. So we wrap the function inside useMemo. Now the function will run only when the states inside the dependency array change, and it will not run on other state changes.
useMemo remember a value ,so React does not calculate it again and again.
It is used for performance optimization
if we have a component and we have two states in components, and one state we are updating inside a funtion , now when we update or change any state our function is running, we want that our function should run on our order, because running of function on every state changing effect the performance of our component, so we use useMemo hook and keep the function inside useMemo hook and we can order to run our function according to the state change that we give inside useMemo hook dependecy.
useMemo hook takes a callback function inside its parameter,and a dependency array, we write functionality inside funciton blocks { } and in dependecy array we write the state on which we want to run our function.

üí° Example



 import { useMemo, useState } from "react";

function UseMemo() {
  const [number, setNumber] = useState(0);
  const [count, setCount] = useState(0);

  const doubleNumber = useMemo(() => {
    console.log("Calculating...");
    return number * 2;
  }, [number]);   // Only re-run when "number" changes

  return (
    <div>
      <h1>Number: {number}</h1>
      <h2>Double: {doubleNumber}</h2>

      <button onClick={() => setNumber(number + 1)}>Increase Number</button>
      <button onClick={() => setCount(count + 1)}>Increase Count</button>
    </div>
  );
}

export default UseMemo;

The component renders before the API response arrives

Question 35‚ùì

Q: Why we use cleanup in react?

‚úÖ Answer

When a page uses setTimeout and the user navigates to another page before the time completes, the timeout still runs and the API call happens even though the page is gone.
Cleanup is used to remove the previous page‚Äôs activities (like timers or listeners) when the component unmounts, so unnecessary API calls don‚Äôt happen and app performance stays fast.
