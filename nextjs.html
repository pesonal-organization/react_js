Question 2❓

Q: types of components in nextjs?

✅ Answer

two types of components are in nextjs

  • server components
  • client component

    • Question 3❓

    Q: what is server components?

    ✅ Answer

    server components are components that render on server side and the html we write in server components , is send to browser but the JavaScript that we write in server components is not sent to browser and user only see html code output and he can not see JavaScript code out put on browser.

    all app routes components of next js are server components.

    Question 4❓

    Q: client side components in next js?

    ✅ Answer

    client side components are rendered on browser and their JavaScript code also send to browser and user can see the output/functionality of JavaScript code also on browser.

    Question 5❓

    Q: how can we make server component to client component?

    ✅ Answer

    in the top of component write 'use client'
    the server component is converted from server to client component.

    Question 6❓

    Q: when we need to make server component to client component?

    ✅ Answer

  • when we use any hook in our component like
    useEffect,useState,useRef,useContext etc
  • when we use onclick handlers like
    onclick,onChange,onSubmit
  • when we create any thing that can give response on user action
  • Add to cart button
  • Image slider
  • login/sign up form
  • cart count page that updates

    • Question 7❓

    Q: when we use server component?

    ✅ Answer

    we use server components when

    Data Fetching

    when we are fetching data by api calling

    Static/not dynamic needed

    when we have to shown only data/ui not need any event like click etc

    secret and sensitive data

    apiKeys,token, secret data keep only on server component

    Prouducts listing page

    when we only need to show all Prouducts

    Proudct details page dikhana ho

    Navbar only links

    Question 13❓

    Q: what is NextRequest and NextResponse?

    ✅ Answer

    we import NextRequest and NextResponse from next/server

    NextRequest

    in website when user want to access private or publicroutes, the user request for getting private/public routes, users request goes through NextRequest that is defined in proxy.ts file to the server , server checks the requirement for giving respose for the request of user if requirements are fullfilled than server give respose by

    NextResponse

    to the user. This all process are handled by proxy or middleware

    NextResponse

    the answer that comes from server after sending request from browser using NextRequest is called NextResponse.

    Question 14❓

    Q: what is req.nextUrl.pathname?

    ✅ Answer

    give us location on which we are now / give us current route name on which we are standing now like
    /about, /conatct, / etc

    💡 Example

    
       

  //proxy.ts
import { NextRequest,NextResponse } from "next/server";
import { publicRoutes,privateRoutes } from "./src/routes/routes";

const proxy = (req:NextRequest)=>{
    const cookiesVal = req.cookies.get('myToken')?.value;
    console.log('cookies value ',cookiesVal)
    const path = req.nextUrl.pathname;
    console.log('checking ',req.nextUrl.pathname)
    console.log('path ',path);
    console.log('full url ',req.url)
    if(cookiesVal && publicRoutes.includes(path)){
        console.log('public routes are accessing')
        return NextResponse.redirect(new URL('/',req.url))
    }
if(!cookiesVal && privateRoutes.includes(path)){
    console.log('accessing private routes')
    return NextResponse.redirect(new URL('/login',req.url))
}
    
}
export default proxy;

    Question 17❓

    Q: what is setCookie?

    ✅ Answer

    setCookie is a method of cookie that is used to store token in cookie storage
    we import it from 'cookies-next'
    to import it first we have to install 'cookies-next' in our project by using following comman
    npm i cookies-next

    💡 Example

    
       
        'use client'
import React, { useState } from 'react'
import {auth} from '@/src/utils/firebase';
import { signInWithEmailAndPassword } from 'firebase/auth';
import {setCookie} from 'cookies-next'
const LogInPage = () => {
  const [email,setEmail] = useState('');
  const [password,setPassword] = useState('')
  //logInHandler
  const logInHandler = async()=>{
    try{
      const saveUser = await signInWithEmailAndPassword(auth,email,password)
      alert('sign in successfully')
      console.log(saveUser)
      //get tokeId
      const tokenVal = await saveUser.user.getIdToken();
      console.log('token ',tokenVal)
      //storing tokenVal in cookies
      setCookie('myToken',tokenVal,{maxAge: 60*60*3})

      window.location.reload()
     

    }
    catch(error){
      console.log(error)
    }
    
  }
  return (
    <>
    <h1>Login</h1>
    <input type="email" placeholder='enter your email'
    autoComplete='new-email'
    onChange={(e)=>setEmail(e.target.value)}
    value={email} />
<br/>
    <input type="password" placeholder='enter your password'
    autoComplete='new-password'
    onChange={(e)=>setPassword(e.target.value)}
    value={password} />
    <br/>
    <button onClick={logInHandler}>Login</button>
    </>
  )
}

export default LogInPage

    Question 18❓

    Q: what is server actions?

    ✅ Answer

    when we call an api in client component , we can see our api Key and any one can access our api key and he can use our key and every server on which we deploy our backend data , give us only request limits for accessing our data,
    when our key is shown in network tab , any one can access it and he can use it and he can finish our api request limit,

    Now it is question that how can we hide our api key that anyone can not access it???

    it problem is solved by next js by server actions when we call any api using sever actions , next js hide our api key by default and give a dummy api key so our key is not accessible to any one.

    using of server actions

    we create a file actions.ts in our app folder main directory with layout.tsx file , we can't keep this file name except actions.ts any another.
    we make an async await function in actions.ts file and call an api and return api data when api come and we export this function and in page we need to use api we import this functions and we create an async await function inside our component and we store this export funciton in variable that we define in componets' s function and we can use it.

    💡 Example

    
       
        //actions.ts
'use server'
const fetchAllNewUseres = async()=>{
    const apiUrl = 'https://jsonplaceholder.typicode.com/users';
    const res = await fetch(apiUrl);
    const jsonData = await res.json();
    return jsonData;
}
export {fetchAllNewUseres} 
______________________ 
we use this api in our page 
see following code 

'use client'
import React, { useEffect } from "react";
import { fetchAllNewUseres } from "../actions";
const NewPage = ()=>{
    const getUsers = async()=>{

        console.log('users by action file ', geti)

    useEffect(()=>{
       getUsers()
    },[])

    return(
        <>
        <h1>I am new Page</h1>
        </>
    )
}
export default NewPage;
