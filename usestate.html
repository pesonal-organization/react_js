Question 1‚ùì

Q: what is useState?

‚úÖ Answer

state is a variable and it is used to store data, it stores information about the component.
when we store a data and variable and makes any changes in it , the data or changes not change on screen
BUT
when we store any data in state and make any change in state , the changes appears on screen also.
KEY FEATURES:
1. Used to store data in a component.
2. Change in state refresh the UI automatically.
3. We use useState hook in functional components.
USE OF useState hook
1.
import at start of component
import {useState} from 'react';
2.
syntax of useState
const [var, setVar] = useState(initialValue)
Example:
const [city,setCity] = useState('Karachi')

üí° Example


       
        usestate.jsx 

        import React, {  useState } from 'react'

const StateInReact = () => {
  const [city,setCity] = useState('Karachi')
  // change city
  const changeCity = ()=>{
    setCity('Islamabad')
    
  }
  return (
   <>
    <div>StateInReact</div>
    <h1>{city}</h1>
    <button onClick={changeCity}>change city</button>
   </>
  )
}

export default StateInReact

Question 2‚ùì

Q: what is hooks in ReactJs?

‚úÖ Answer

Hooks are predefined/built-in-function and we use multiple hooks and each hook has specific duty.
Short History
Hook were added in ReactJs version 16.8 in 2019 and it was allowed to use hooks in functional components
Before this state and sideeffcets are used only in class components.
Hooks are only used in functional components not in class components.
Important Hooks:
1. useState hook
useState hook is used for state management
2. useEffect hook
useEffect hook is used for handling side effects like API calls
3. useContext Hook
useContext hook is used for context
4. useRef Hook
useRef hook is used for accessing DOM or keeping values
5. useMemo
useMemo hook is used for memoization.
6. useCallback hook
useCallback hook is used to prevent function re-creation.

Question 8‚ùì

Q: what is useEffect hook?

‚úÖ Answer

useEffect is a Hook in React that allows you to run side effects in your functional components.
syntax of useEffect

useEffect((function(),[dependencies]))
dependencies are optional
useEffect(()=>{
        code for side effects},[dependencies])

üí° Example


       
        useeffect.jsx 

        import React, { useEffect, useState } from 'react'

const Effects = () => {
  const [count,setCount ] = useState(1);
  const [data,setData] = useState(5)
  function oneCall(){
    console.log('counter: ',count, '\t','data : ',data)
  }
useEffect(()=>{
    oneCall();
},[])
  return (
  <>
  <h1>useEffect in reactJs</h1>
  <h1>counter : {count}</h1>
  <h1>data: {data}</h1>
  <button onClick={()=>setCount(count + 1)}>counter</button>
  <button onClick={()=>setData(data + 5)}>update data</button>
  </>
  )
}

export default Effects

Question 9‚ùì

Q: what is sideEffects?

‚úÖ Answer

Side effects are actions that happen outside the main function of rendering UI.
These actions don't directly return UI but affect the app's behaviour or system.

useRef hook is use to accessing any DOM element and for controlling any html element by react component.

const UseRefHook = () => {

      inputName.current.style.display = 'none'
    }
  }
// para
const paragraphAccessing = ()=>{
  console.log(para.current)
} 

  return (
    <>
    <h1>useRef hook in reactJs</h1>

    <p ref={para}>I am learning reactjs</p>

    <button onClick={hideShow}>Hide</button>
